# 账号使用指南

## 1. 这套系统怎么用

这个项目不是直接登录 ChatGPT 网页来聊天，而是把某个账号的 `api_key` 或 `access_token` 挂到本地代理：

- 本地代理地址：`http://127.0.0.1:8000/v1`
- 客户端只连本地代理
- 后端根据“当前激活账号”决定用哪个凭证去请求上游

适用场景：

- 给 Codex / OpenAI 兼容客户端统一切号
- 管理多组账号和 token
- 用域名邮箱批量注册并补抓 token

## 2. 账号“能用”是什么意思

对这个项目来说，一个账号要想真正可用，至少要满足下面两点：

1. 账号里有可用的 `api_key` 或 `access_token`
2. 这个凭证对上游接口有权限，至少不能在 `GET /v1/models` 时返回权限错误

如果只有 token 但没有上游权限，那么：

- 管理后台会显示“已配置 token”
- 但真正走代理调用时仍然会失败

## 3. 当前账号测试结果

已测试库中 6 个账号，测试方式为：

1. 逐个切换为激活账号
2. 通过项目代理调用上游 `GET /v1/models`
3. 观察返回状态码和错误信息

结果：

- 6/6 均返回 `403 Forbidden`
- 上游错误为缺少 `api.model.read` 权限
- 说明这些账号的 `access_token` 目前不满足这个项目的代理使用条件

结论：

- 这些账号“已录入、有 token”
- 但“目前不能直接拿来走本项目的 Codex / OpenAI 兼容代理”

## 4. 正确使用步骤

### 启动项目

Windows:

```bat
run_backend.bat
run_frontend.bat
```

默认地址：

- 后端：`http://127.0.0.1:8000`
- 前端：`http://127.0.0.1:5173`

### 在前端里检查账号

进入：

- `账号管理`
- `Token 管理`
- `Codex 集成`

重点看：

- 是否存在 `API Key` 或 `access_token`
- 切换账号后，`Codex 集成` 页面是否显示已配置激活账号

### 让客户端走本地代理

PowerShell:

```powershell
$env:OPENAI_API_BASE="http://127.0.0.1:8000/v1"
```

CMD:

```bat
set OPENAI_API_BASE=http://127.0.0.1:8000/v1
```

Bash:

```bash
export OPENAI_API_BASE="http://127.0.0.1:8000/v1"
```

然后你的 Codex 或 OpenAI 兼容客户端就会把请求发到本地代理。

## 5. 怎么判断新账号可不可用

推荐按这个顺序检查：

1. 在 `账号管理` 看账号是否已录入
2. 在 `Token 管理` 看是否有有效 token
3. 在 `Codex 集成` 切换到该账号
4. 用代理请求一次 `GET /v1/models`

只要这一步返回：

- `200`：通常说明代理链路可用
- `401`：凭证失效或错误
- `403`：有凭证，但权限不足
- `503`：当前激活账号没有可用凭证

## 6. 你这批账号当前为什么不能用

这次测试返回的是：

```text
403 Forbidden
Missing scopes: api.model.read
```

这通常意味着：

- token 被识别了，但不是这个代理场景需要的权限集
- 账号没有对应 API 访问权限
- 组织 / 项目角色不足
- 或拿到的是不能直接访问当前 OpenAI API 的凭证

## 7. 后续建议

如果你要继续用这套项目，建议优先这样做：

1. 先找一个确认可调用 OpenAI API 的 `api_key`
2. 把它录入某个账号并设为激活账号
3. 再用 `http://127.0.0.1:8000/v1` 做联调
4. 现有这批仅有 `access_token` 的账号，不要直接当成可用 API 账号

## 8. 自测命令

后端启动后，可以直接测：

```powershell
curl http://127.0.0.1:8000/api/codex/status
```

再测代理：

```powershell
curl http://127.0.0.1:8000/v1/models
```

如果你要继续排查账号问题，优先关注返回的 `401 / 403 / 503`。

## 9. 插件登录态切号

从当前版本开始，`Codex 集成` 页面里有两种不同的切换：

- `切换代理激活账号`：只影响本地 `http://127.0.0.1:8000/v1` 代理
- `切换插件登录账号`：直接写入本机 `~/.codex/auth.json`

第二种才是真正切 VS Code Codex 插件显示的登录账号。

切换插件登录态的前提：

1. 账号必须有 `access_token`
2. 账号必须有 `refresh_token`

切换后如果插件侧边栏没有马上刷新，执行：

```text
Developer: Reload Window
```

说明：

- 该功能会先备份旧的 `auth.json`
- 当前实现主要依赖 `access_token + refresh_token`
- 如果缺少匹配的 `id_token`，通常仍可用，但界面信息可能需要重载后才完全刷新
